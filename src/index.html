JSX 1|10

Was ist JSX?

  • Syntaxerweiterung für Javascript
  • HTML-ähnlichen Code innerhalb von JavaScript
  • 2011 von Meta(Facebook) entwickelt

JSX steht für JavaScript XML und ist eine Syntaxerweiterung für JavaScript, welche es ermöglicht "HTML" Code innerhalb von JavaScript zu schreiben. Es wird als sogenannter Syntax-Sugar bezeichnet
JSX wurde von Meta (früher Facebook) entwickelt, um die Integration von HTML und JavaScript in React-Anwendungen zu erleichtern.

JSX 2|10

Wie funktioniert JSX?

  • Umwandlung in JS bevor es ausgeführt wird
  • Verwendung von Transpiler (Babel)
  • Funktionsaufrufe

Der Code wird vor der Ausführung im Browser von einem Transpiler, wie Babel, in JavaScript umgewandelt. Für die Umwandung wird ein Transpiler verwendet. Der beliebteste Transpiler für JSX ist Babel.
Babel transformiert JSX-Code in eine Reihe von Funktionsaufrufen, die den ursprünglichen HTML-ähnlichen Code entsprechen. Der Browser kann dann den resultierenden JavaScript-Code ausführen.

JSX 3|10

Syntax

  • Es darf nur ein Wurzelelement existieren
  • ClassName statt Class verwenden
  • Geschweifte Klammern für JavaScript-Ausdrücke
  • Tags müssen immer geschlossen werden

In JSX darf es immer nur ein Wurzelelement geben, welches mit return zurückgegeben wird. Das bedeutet das der gesammte JSX Coder einer Komponente in einem einzigen äußeren Element platziert sein muss.
Um Nameskonflikte zu vermeiden, wird in JSX className anstatt class verwendet, da class ein reserviertes schlüsselwort in JavaScript ist.
Innerhalb von geschweiften Klammern können beliebige JS Ausdrücke kombiniert werden.
